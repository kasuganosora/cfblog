# CFBlog 项目修复任务清单

## 问题概述

项目存在多个严重问题需要修复：
1. 修复validateSessionID/后台登录态问题
2. validateSessionID 有多个版本（admin.js 和 user.js 各一个），需要整合,抽取所有组件里的公共方法到公共库中华
3. 后台功能因登录态问题无法验证
4. E2E测试用例需要重写（之前测试显示通过，但人工测试登录都失败）
5. 前台侧栏链接404错误
6. 前台风格未按照样板网站设计
7. 前台功能可用性未知（因后台问题无法测试）
8. 需要优化前台接口静态化到R1
9. 需要添加Markdown编辑器


## 详细任务清单

### 1. 代码重构与整合
- [ ] **整合validateSessionID到common.js**
  - 当前admin.js和user.js中都有validateSessionID函数
  - 需要创建common.js或utils.js统一管理公共函数
  - 检查其他重复的函数并整合

### 2. 后台功能修复
- [ ] **修复后台登录态验证问题**
  - 检查session验证逻辑
  - 验证环境变量SESSION_SECRET设置
  - 检查数据库连接和用户权限验证
  - 确保后台功能可用性

### 3. 测试用例重写
- [ ] **重写E2E测试用例**
  - 基于实际功能重写测试用例
  - 确保测试能真实反映系统状态
  - 添加登录测试、功能测试等
  - 修复测试覆盖率问题

### 4. 前台功能修复
- [ ] **修复前台侧栏404链接**
  - 检查所有侧栏链接
  - 修复路由配置
  - 确保所有链接指向正确的页面

- [ ] **按照样板网站重新设计前台风格**
  - 分析样板网站的设计风格
  - 重新设计UI组件
  - 统一颜色、字体、间距等设计规范

- [ ] **测试前台功能可用性**
  - 在后台修复后测试前台功能
  - 验证文章浏览、搜索、评论等功能
  - 测试响应式设计

### 5. 性能优化
- [ ] **优化前台接口静态化到R1**
  - 识别可静态化的接口（如文章列表（文章列表在未搜索的时候自动使用R1上静态化的json）、分类，tag等）
  - 实现R1缓存策略
  - 减少D1和KV的使用量
  - 设置合理的缓存过期策略

### 6. 功能增强
- [ ] **添加Markdown编辑器到文章编辑器**
  - 集成Markdown编辑器组件
  - 支持实时预览
  - 添加常用Markdown语法快捷方式
  - 支持图片上传和插入

3. **低优先级**（后续优化）：
